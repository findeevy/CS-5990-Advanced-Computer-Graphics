vk::Extent2D SwapChain::chooseSwapExtent ( const vk::SurfaceCapabilitiesKHR &  capabilities)
private

Chooses optimal swap extent based on surface capabilities.

Chooses the swap chain image extent based on surface capabilities.

Explanation:

  • Some window systems (e.g., Android) provide a fixed currentExtent; in that case the driver requires you to use that exact extent.
  • If currentExtent has the special value of UINT32_MAX, the surface allows the app to choose an extent. We fall back to a default (800x600) and clamp it to the supported min/max extents provided by the surface capabilities.
Parameters
capabilitiesThe surface capabilities retrieved from the physical device.
Returns
The chosen vk::Extent2D extent for swap chain images.

vk::PresentModeKHR SwapChain::chooseSwapPresentMode ( const std::vector< vk::PresentModeKHR > &  availablePresentModes)
private

Selects the best available presentation mode for rendering.

Chooses the preferred swap chain present mode.

Present modes:

  • eMailbox: low latency, no tearing (if supported); similar to triple buffering.
  • eImmediate: may present images immediately and can tear (not recommended).
  • eFifo: guaranteed to be available on all platforms; behaves like vsync.

We prefer mailbox when available, otherwise fallback to FIFO for portability.

Parameters
availablePresentModesList of supported present modes returned by the driver.
Returns
The chosen vk::PresentModeKHR (mailbox preferred, otherwise FIFO).

Selects the preferred surface format (color space and pixel format).

Chooses the optimal swap chain surface format.

Preferred format:

  • VK_FORMAT_B8G8R8A8_SRGB with sRGB nonlinear color space for correct color reproduction on most platforms.

If that specific format is not available we return the first available option. In a more advanced implementation you could score formats based on color depth, linear vs sRGB, and device support.

Cleans up swap chain resources before destruction or recreation.

Cleans up swap chain resources.

Originally Vulkan::cleanupSwapChain()

Using RAII containers means most cleanup is done by destructors, but we still:

  • Clear the vector of image views (explicitly destroying the RAII view objects).
  • Reset the swapChain RAII handle to release the underlying VkSwapchainKHR.

This is called before creating a new swap chain or when shutting down the renderer.

◆ create()

void SwapChain::create ( )

Creates the Vulkan swap chain and associated image views.

Creates the Vulkan swap chain and its associated image views.

Originally Vulkan::createSwapChain()

High-level steps:

  1. Query surface capabilities and available formats/present modes from the GPU.
  2. Choose a surface format, present mode, and extent (resolution).
  3. Build a vk::SwapchainCreateInfoKHR describing how the swap chain should behave.
  4. Construct the RAII-wrapped vk::raii::SwapchainKHR object.
  5. Retrieve raw swap chain images and create RAII-wrapped image objects so we can create corresponding image views (one per swap chain image).

Important notes:

  • This function can throw on Vulkan errors (vk::SystemError) when creating the swapchain or image views. Callers are responsible for handling exceptions appropriately.
  • RAII objects hold references to the device; do not destroy the device before the swapchain.
Exceptions
vk::SystemErrorif swap chain or image view creation fails.

void SwapChain::recreate ( )

Recreates the swap chain, typically after window resizing or surface loss.

Recreates the swap chain (e.g., after a window resize event).

Originally Vulkan::recreateSwapChain()

Typical usage pattern:

  • On framebuffer resize, call recreate() to re-query surface capabilities and rebuild the swap chain and image views to match the new window size.

Note: More robust implementations may need to wait for the device to become idle or to free other resources (framebuffers, pipelines) that depend on the swapchain.
